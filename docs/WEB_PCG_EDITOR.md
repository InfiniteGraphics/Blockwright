# Blockwright Web PCG Editor

Blockwright now hosts the PCG editor as a static React Flow application served by the Minecraft client over a localhost bridge.

## Architecture

```text
Browser static editor
  React + @xyflow/react canvas
  TypeScript PCG graph runtime
  graph JSON compiler / validator
        ↓ HTTP JSON
Minecraft client localhost bridge
        ↓ C2S packets
Minecraft server / integrated server
  Java PcgGraphExecutor is authoritative for world preview and bake
```

The web editor is intentionally split into two layers:

1. `web/src/runtime`: Blockwright's TypeScript PCG graph runtime. It normalizes graph JSON, applies node defaults, validates ports, detects cycles, computes execution order, and produces diagnostics.
2. `web/src/components`: React Flow UI. It is only responsible for editing, inspecting, and visualizing the graph.

The Java runtime remains authoritative for anything touching the world. The TypeScript runtime is an editor-side compiler and diagnostic layer so the user can catch graph mistakes before sending work to the server.

## Source layout

```text
web/
  package.json
  vite.config.ts
  src/
    App.tsx
    bridge/blockwrightBridge.ts
    components/
    runtime/
    styles/app.css

common/src/main/resources/assets/blockwright/web/
  index.html
  assets/*.js
  assets/*.css
```

The files under `common/src/main/resources/assets/blockwright/web` are the compiled static bundle served by `BlockwrightWebBridge`. They are checked in so normal mod builds can run without Node.js.

## Rebuilding the static bundle

```bash
cd web
npm ci
npm run build
```

or from Gradle:

```bash
./gradlew :common:buildWebEditor
```

The Vite build writes directly to:

```text
common/src/main/resources/assets/blockwright/web
```

## Bridge API

The browser receives a random token in the editor URL. API requests include the token as both query string and `X-Blockwright-Token` header.

```text
GET  /api/session
GET  /api/node-schemas
POST /api/session/select
POST /api/graph/preview
POST /api/graph/save
POST /api/graph/bake
```

`/api/node-schemas` is generated by the Java `PcgNodeSchemaRegistry`, so the React Flow editor does not hard-code Java node types.

## Runtime boundary

The TypeScript runtime compiles and validates the graph, but it does not place Minecraft blocks. Preview and bake still flow through the Java server-side `PcgGraphExecutor`.

This gives us UE-style editor ergonomics while keeping Minecraft world mutation server-authoritative.
